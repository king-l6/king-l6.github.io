单页面卡法应用注意事项（1）

一. *.vue 是单文件组件

一个文件即为一个组件，同时也是一个ES的模块
文件中包含三部分： template script style
SPA（单页面应用）不利于搜索因此的SEO(搜索引擎优化)操作
特别是百度根本没办法抓取到SPA的内容页面

二.页面级组件

创建在views文件夹中，而且，因为页面级组件都要实现跳转，所以是一级路由组件

三、注册路由

大部分vue的第三方模块在引入后，需要用Vue.use进行注册

四、路由配置

是全局的，在整个单页面应用的跳转中都是要匹配路由配置的。

五、路由配置中的模式（mode）

路由配置中的模式（mode）有两种： hash(有#)   history (没#)
默认的mode为hash，如果想要使用history，就需要给mode添加history
在Router - index.js中修改配置

面试题：路由的hash和history模式的区别

  1. 首先hash模式url带#号，history不带#号
  2. hash模式前端路由修改的是hash值(#及以后)，对后端没影响，因此改变hash也不会重新加载页面，比如修改为了不存在的#123页面，页面不会跳转;
    history模型刚好相反，没有对应的页面就会出现404

十、scoped的使用

scoped 使当前组件的样式不会和其它组件冲突
使用 scoped 后，父组件的样式将不会渗透到子组件中。不过一个子组件的根节点会同时受其父组件的 scoped CSS 和子组件的 scoped CSS 的影响。这样设计是为了让父组件可以从布局的角度出发，调整其子组件根元素的样式

面试题：面试的时候也经常会被问到scoped
使当前组件的样式不会和其他组件冲突；
原理就是在css样式的后面添加了自定义属性
