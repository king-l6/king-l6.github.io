Vue组件/组件通讯

课程目标

  1. 了解组件的应用场景
  2. 理解组件的运行机制
  3. 掌握组件的定义和使用

1. 认识组件

通常一个应用会以一棵嵌套的组件树的形式来组织

例如，你可能会有页头、侧边栏、内容区等组件，每个组件又包含了其它的像导航链接、博文之类的组件。为了能在模板中使用，这些组件必须先注册以便 Vue 能够识别。
组件特点

  1. 组件（Component）是 Vue.js 最强大的功能之一；
  2. 组件可以扩展 HTML 元素，封装可重用的代码
  3. 组件系统让我们可以用独立可复用的小组件来构建大型应用，几乎任意类型的应用的界面都可以抽象为一个组件树

这里有两种组件的注册类型：

  • 全局注册
  • 局部注册

你可以将组件进行任意次数的复用

注意：当点击按钮时，每个组件都会各自独立维护它的 count。因为你每用一次组件，就会有一个它的新实例被创建。
注意事项：

  1. 一个组件的 data 选项必须是一个函数，因此每个实例可以维护一份被返回对象的独立的拷贝
  2. 组件的模板中只能有一个根元素

全局注册的组件可以用在其被注册之后的任何 (通过 new Vue) 新创建的 Vue 根实例，也包括其组件树中的所有子组件的模板中。

2.1.4 总结

全局组件，在任意一个vue的挂载点中都能直接使用
注意事项：
1. 定义全局组件，组件的名称首字母要大写（在模板中引用时要转为小写，且多个单词中间加‘-’）
2. 组件的配置：组件的配置和vue实力话的配置基本一致，同样都有data，methods，computed，生命周期函数；不同之处，没有el，一定有template
3. 组件中不能访问根实例中的数据
4. 组件中的data不是对象，而是一个有返回值的函数
5. 组件的模板中只能有一个根元素【否则报错】

总结：

使用函数反对对象，这样每一个实例的data属性都是独立的，不会相互影响了

3. 组件的应用

3.1 组件应用中的注意事项

  • 组件名首字母大写，在模板中应用时要转为小写，如果是两个单词，中间用”-“连接
  • template模板中只能有一个根元素
  • 子组件中的data必须是一个有返回值的函数

组件是 vue.js最强大的功能之一，而组件实例的作用域是相互独立的，这就意味着不同组件之间的数据无法相互引用。一般来说，组件可以有以下几种关系：

针对不同的使用场景，如何选择行之有效的通信方式？这是我们所要探讨的主题。本文总结了vue组件间通信的几种方式，如props、$emit/$on（bus总线）、vuex
我们在这里重点说的就是，父子组件之前的传值

4.1父传子【props】

父组件通过 Prop 向子组件传递数据

  1. 父组件通过自定义属性将msg的值传给子组件
  2. 子组件通过 props接收父组件的值

父组件

子组件（com）

注意：在父向子组件传值时，如果父组件的值发生变化，子组件也会变化，但是子组件改变，不会引入父组件改变，我们称之为时“单向数据流”

总结：

  1. 父组件通过自定义属性，向子组件传递数据
  2. 子组件通过props接收传递的数据
  3. 当父组件的值发生变化时，子组件也会改变
  4. 尽量避免通过子组件更改父组件传递来的数据
  5. 子组件可以改变父组件传递来的引用数据类型，数据改变，父组件也会改变
  6. 属性名不能有大写字母
